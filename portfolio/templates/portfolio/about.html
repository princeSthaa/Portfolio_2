How I Work

I approach every project with a problem-first mindset: before writing a single line of code, I try to deeply understand the "why." What problem are we solving? Who is the user? What does success look like?

I prefer iterative development — ship a lean version, gather feedback, and improve. I'm a big fan of test-driven development for critical logic, good documentation, and meaningful code reviews.

My workflow typically involves Notion for planning, VS Code as my editor, Git + GitHub for version control, and Linux as my daily driver OS.
